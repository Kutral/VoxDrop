# VoxDrop Project Reference

## Snapshot

- Project: `VoxDrop`
- Type: Windows-first desktop dictation app
- Stack: Tauri, Rust, React, TypeScript, Vite, Tailwind CSS
- Purpose: capture speech with a global hotkey, transcribe and clean it with Groq, expand snippets, then paste the final text into the currently focused app

## Repository Scope

This document describes the project files that belong in the GitHub repository. Local agent folders, editor-only personal config, dependency installs, and Rust build output are intentionally excluded from version control.

## High-Level Architecture

### Frontend

- `src/App.tsx`: switches between the main dashboard window and the pill window
- `src/components/MainView.tsx`: history, snippets, settings, and update UI
- `src/components/PillView.tsx`: listening and processing pill experience
- `src/lib/groq.ts`: Groq transcription, cleanup, and API-key validation
- `src/lib/updates.ts`: checks GitHub releases for newer versions
- `src/store.ts`: persisted local state for settings, snippets, and history

### Native backend

- `src-tauri/src/lib.rs`: boots Tauri, wires commands, windows, events, and startup behavior
- `src-tauri/src/audio.rs`: records microphone input and returns WAV audio
- `src-tauri/src/paste.rs`: writes to clipboard and simulates paste
- `src-tauri/src/db.rs`: SQLite migrations
- `src-tauri/src/windows_hotkey.rs`: Windows low-level keyboard hook for modifier-based dictation hotkeys
- `src-tauri/src/main.rs`: binary entrypoint

## Recent Milestones

- **Audio Earcons (March 2026)**: Added zero-latency Web Audio API synthesized earcons. `playStartEarcon` triggers a crisp upward sweep on hotkey press for immediate feedback, and `playSuccessEarcon` plays a reassuring soft chime upon successful text paste.
- **Pasting Optimization (March 2026)**: Re-implemented instant clipboard-based pasting via the Win32 API. Utilizes the `ExcludeClipboardContentFromMonitorProcessing` format to bypass the Windows Clipboard History while offering significantly faster pasting than direct keystroke simulation (`enigo`).
- **v0.0.6 Release (March 2026)**: Completely rewrote the hotkey engine to support chorded non-modifier combinations (like `N+M`). Fixed a state-sync race condition in the Settings UI for reliable hotkey recording.
- **Performance Optimization (March 2026)**: Drastically reduced hotkey latency by moving audio recording start and system muting to the Rust backend. Implemented persistent audio stream pre-warming to achieve <50ms response time.
- **V0.0.4 Release**: Initial project structure with Groq integration and Windows-first support.

## Directory Notes

### `.github/workflows/`

- README visuals used for the banner and architecture image

### `public/`

- static public assets served by Vite
- currently contains the project favicon only

### `src/`

- React frontend source

### `src/components/`

- dashboard and pill window UI

### `src/lib/`

- frontend utility modules for Groq integration and release checks

### `src-tauri/`

- native desktop project, packaging config, icons, and Rust source

### `src-tauri/icons/`

- platform packaging icons for Windows, macOS, iOS, and Android targets generated by Tauri tooling

## Commands

- `npm install`: install frontend dependencies
- `npm run dev`: run the Vite web dev server only
- `npm run build`: build the web frontend
- `npm run tauri dev`: run the full desktop app in development mode
- `npm run tauri build`: build the production desktop installer/bundles

## API and Settings

- API provider: Groq
- API key sources:
  - `Settings -> Neural API Key`
  - `.env.local` via `VITE_GROQ_API_KEY`
- Persisted locally:
  - API key
  - models
  - hotkey
  - snippets
  - activity history

## Ignored Local-Only Content

The repository intentionally excludes these local-only paths:

- `.agents/`
- `.vscode/`
- `node_modules/`
- `dist/`
- `src-tauri/target/`
- local `.env` files other than `.env.example`
- local SQLite files
